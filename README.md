# 🚀 StartFlow

**StartFlow** — это система для управления проектами и задачами с поддержкой AI-аналитики. Проект помогает командам организовывать рабочие процессы, вести задачи, отслеживать состояние проекта и получать сгенерированные описания, отчеты и аналитические выводы на основе данных проекта.

Основная идея StartFlow — объединить классический таск-менеджер с AI-помощником, который помогает быстрее понимать контекст проекта, анализировать прогресс команды и формировать текстовые материалы для отчетности.

## ✨ Основные возможности

* 📁 создание и управление проектами;
* ✅ работа с задачами, колонками и исполнителями;
* 🧩 выбор готовых карточек с разделами задач под разные типы проектов;
* 🔐 авторизация и управление пользователями;
* 📌 формирование проектных досок;
* 🤖 генерация AI-описания проекта для быстрого погружения новых участников;
* 📊 генерация отчетов по проекту за выбранный период;
* 📈 генерация аналитических ответов и данных для графиков на основе проектной информации.

## 🏗️ Архитектура проекта

Проект состоит из нескольких основных частей:

### 🖥️ Frontend

Клиентская часть отвечает за пользовательский интерфейс: страницы авторизации, список проектов, создание проектов, проектные доски, аналитику и взаимодействие с AI-функциями. При создании или настройке проекта пользователь может выбирать готовые карточки с разделами задач, которые подходят для определенных типов проектов и помогают быстрее сформировать рабочую структуру доски.

### ⚙️ Backend

Основной backend-сервис отвечает за работу с пользователями, проектами, задачами, участниками проектов, колонками и связанной бизнес-логикой. Также в backend реализована работа с базой данных и миграциями.

### 🧠 AI Model Service

Отдельный AI-сервис используется для генерации текстов и аналитики по проектным данным. Он принимает документы с описанием проекта и задачами, после чего формирует ответы с учетом переданного контекста.

AI-сервис поддерживает генерацию:

* 📝 общего описания проекта;
* 📄 отчета по проекту за период;
* 💬 текстовых аналитических ответов;
* 📊 структурированных данных для графиков.

## 🛠️ Используемый стек

В проекте используются:

* **Frontend:** Svelte / SvelteKit, JavaScript, TypeScript;
* **Backend:** Python, FastAPI, SQLAlchemy, Alembic;
* **AI-сервис:** Python, FastAPI, llama-cpp-python, локальная GGUF-модель Vikhr;
* **Инфраструктура:** Docker, Docker Compose;
* **База данных:** PostgreSQL.

## 🤖 AI-функциональность

AI-модуль работает с проектными данными: описанием проекта, задачами, статусами, дедлайнами, исполнителями и другой связанной информацией. На основе этих данных модель может сформировать понятное описание проекта для нового участника команды или отчет о работе за определенный период.

AI-сервис не должен выдумывать отсутствующие факты: генерация строится только на переданных документах и задачах. Это позволяет использовать его для проектной отчетности, onboarding-сценариев и аналитики состояния проекта.

## 🎯 Цель проекта

StartFlow разрабатывается как инструмент, который помогает командам не только вести задачи, но и быстрее понимать, что происходит в проекте. Система должна снижать нагрузку на участников команды при подготовке отчетов, описаний и аналитических материалов, используя данные, которые уже есть внутри проекта.
