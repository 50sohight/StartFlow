# Инструкция

Ниже шаги "скопировал -> запустил -> прислал результат".

## 1) Скачать проект

```bash
git clone <ТВОЯ_ССЫЛКА_НА_РЕПО>
cd StartFlow/ai-model
```

## 2) Установить зависимости

```bash
python -m pip install -r requirements.txt
```

Если `llama_cpp` не установился, поставить prebuilt wheel под свою версию Python и Windows.

## 3) Подключить GGUF модель

Вариант A (рекомендуется): указать абсолютный путь к файлу модели.

PowerShell:

```powershell
$env:STARTFLOW_MODEL_PATH="D:\models\Llama-3.1-8B-Instruct.Q4_K_M.gguf"
```

Git Bash:

```bash
export STARTFLOW_MODEL_PATH="D:/models/Llama-3.1-8B-Instruct.Q4_K_M.gguf"
```

Вариант B: положить модель в `ai-model/models/` и задать имя файла:

PowerShell:

```powershell
$env:STARTFLOW_MODEL_FILENAME="Llama-3.1-8B-Instruct.Q4_K_M.gguf"
```

Git Bash:

```bash
export STARTFLOW_MODEL_FILENAME="Llama-3.1-8B-Instruct.Q4_K_M.gguf"
```

## 4) Выключить mock, чтобы тестировать только реальную модель

PowerShell:

```powershell
$env:STARTFLOW_MOCK_LLM="0"
$env:STARTFLOW_ALLOW_MOCK_FALLBACK="0"
```

Git Bash:

```bash
export STARTFLOW_MOCK_LLM=0
export STARTFLOW_ALLOW_MOCK_FALLBACK=0
```

## 5) Запустить API

```bash
python -m uvicorn app.main:app --host 127.0.0.1 --port 8001
```

Проверка (во 2-м терминале):

```bash
curl http://127.0.0.1:8001/
```

В ответе обязательно должно быть:
`"llm_backend":"llama_cpp"`

Если там `mock` или сервер упал, значит реальная модель не подключилась.

## 6) Прогнать качество промптов

Во 2-м терминале:

```bash
python scripts/run_baselines.py
```

Скрипт проверит 5 кейсов и покажет:

- PASS/FAIL по каждому;
- сырые ответы API;
- итог `X/5`.

Кейсы лежат в `PROMPT_BASELINES.md`.

## 7) Что прислать мне в ответ

1. Вывод `curl http://127.0.0.1:8001/`.
2. Полный вывод `python scripts/run_baselines.py`.
3. Имя GGUF файла, версия Python, ОС.
4. Если что-то не прошло: какой кейс и raw ответ.
